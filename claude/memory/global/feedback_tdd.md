テストを先に書いてから実装する。

**Why:** テストなしの実装を防ぐ。AIは実装を理解した上でテストを書くため、失敗確認のためだけにテストを実行する必要がない。  
alarm プロジェクトで「feat 1件につき fix 3〜7件」の連鎖が発生。原因は StatefulWidget の状態を実装しながら確定していったこと。

**How to apply:**
1. テストを書く
2. 実装する
3. テストを実行する（ここで初めて実行。1回で済む）
4. サブエージェント（implementer等）にも同じ手順を指示する

**UI コンポーネント（StatefulWidget）の追加規則:**
- 実装前に「状態一覧（表示モード・境界条件）」をテストケース名として列挙する
- 境界条件（0件・N件以下・N件以上・全チェック済み等）を必ずカバーする
- ウィジェットテストはシナリオテストと分離して専用ファイルに置く（`*_display_test.dart` 等）
