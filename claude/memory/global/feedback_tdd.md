テストを先に書いてから実装する。

**Why:** テストなしの実装を防ぐ。ただしAIが書く以上、Redステップ（失敗確認のためだけのテスト実行）はトークンの無駄なので省略する。
**How to apply:**
1. テストを先に書く
2. 実装コードを書く
3. テストを実行してパスを確認する（1回だけ）
4. サブエージェント（implementer等）にも同じ手順を指示する
