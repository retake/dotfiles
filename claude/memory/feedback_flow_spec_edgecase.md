フロー仕様（SKILL.md等）を書く際、正常系フローを書いた後に以下の問いで網羅性を確認する：

1. **ファイル・成果物が存在しない場合は？** → フォールバックを定義する
2. **エージェントがESCALATEDを返した場合は？** → どのフェーズに戻るかを明示する
3. **カウントのタイミングはいつか？** → 再実行前か後かを明記する
4. **DONEの判定条件は1通りか？** → 粒度を分けて定義する

**Why:** orchestrateスキルのループレビューで5サイクルかかった。第1〜2回は「分岐がそもそもなかった」構造的欠損、第3〜5回は「あるが粗かった」粒度の詰め。初期実装でエッジケースを書いておけば3回以内に収まった。

**How to apply:** フロー仕様の初稿を書いたら、上記4問を自問してから実装orレビューに渡す。
