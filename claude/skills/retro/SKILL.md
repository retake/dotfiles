# Retro — タスク完了後処理スキル

orchestrate完了後（または手動タスク完了後）に実行する後処理スキル。
振り返りドラフトを恒久ファイルに昇格させ、教訓を蓄積する。

## 実行手順

### ステップ1: ドラフトの確認

1. プロジェクトの `.claude/retrospective-draft.md` を読む
   - 存在しない場合: 「振り返りドラフトが見つかりません。orchestrate完了後に実行してください」と表示して終了する
2. ドラフトから以下を抽出する：
   - **タイトル行**（`# 振り返り: …`）からプロジェクト名・日付を取得
   - **タスクID・完了日時**（メタ情報行）
   - **`_index.mdへの追加候補`** セクションの箇条書き（教訓として追加する）
   - **`アクションアイテム`** セクションの `- [ ]` 未完了項目（自動適用できるものを仕分ける）

### ステップ2: 保存先ファイル名の決定

引数またはタイトルから保存先ファイル名を決定する：

- ファイル名フォーマット: `YYYY-MM-project.md`（例: `2026-04-alarm.md`）
- 引数が指定された場合: `YYYY-MM-{引数}.md` で採番（`date +%Y-%m` で年月取得）
- 引数が省略された場合: タイトル行から推定（推定結果をユーザーに表示してから続行）
- すでに同名ファイルが `~/retrospectives/` に存在する場合: 上書きせず `YYYY-MM-{プロジェクト名}-2.md` で採番する

### ステップ3: ドラフトをコピー

```bash
cp .claude/retrospective-draft.md ~/retrospectives/{ファイル名}
```

コピー後、コピー元（`.claude/retrospective-draft.md`）は削除しない（orchestrate が参照するため）。

### ステップ4: `~/retrospectives/_index.md` の更新

`~/retrospectives/_index.md` を読んで以下を更新する：

**4-1. 教訓サマリへの追記**

`_index.mdへの追加候補` セクションの各箇条書きを「教訓サマリ」に追加する。
- すでに同一または類似の教訓が存在する場合はスキップ（重複確認）
- 追加する箇所: 既存の最後の教訓行の直後

**4-2. 振り返り一覧への追記**

「振り返り一覧」テーブルに新しい行を追加する：

```
| {完了日時の日付部分} | {プロジェクト概要} | [{ファイル名}]({ファイル名}) |
```

- プロジェクト概要はドラフトのタイトル行から抽出する
- すでに同名ファイルへのリンクがある場合はスキップする

### ステップ5: ドキュメント修正の適用

`docs/completion-summary.md` と `docs/design-summary.md` に実装との乖離が記録されている場合、自動適用できるものを適用する。

**対象とする乖離（自動適用可）：**
- `completion-summary.md` の「人間が確認すべき点」に「design-summary.mdの〜を修正」のような指示がある
- `design-summary.md` 内に実装と異なる記述が特定されている（コントラスト閾値・色・API名など）

**適用手順：**
1. `docs/completion-summary.md` の「人間が確認すべき点」を読む（存在する場合）
2. 「design-summary.md の〜」形式の修正指示を抽出する
3. `docs/design-summary.md` 該当箇所をgrepで特定し、Editで修正する
4. 修正した箇所をレポートに含める

**自動適用しない乖離（手動対応が必要なもの）：**
- UIの実装変更を伴うもの（コード修正が必要）
- 判断を要する設計変更

### ステップ6: アクションアイテムの仕分けと報告

ドラフトの `アクションアイテム` を「自動適用済み」「手動対応が必要」に仕分けて表示する。

```
[RETRO] 後処理完了

## 保存先
~/retrospectives/{ファイル名}

## _index.md への追加
教訓 {N}件 追加:
  - （追加した教訓1）
  - （追加した教訓2）
  ...
一覧: 「{プロジェクト概要}」を追加

## ドキュメント修正
（適用した修正の一覧、または「なし」）

## 手動対応が残っているアクションアイテム
- [ ] （未対応のアクション1）
- [ ] （未対応のアクション2）
（なければ「なし」）
```

## 制約

- `.claude/retrospective-draft.md` は削除しない
- `~/retrospectives/` 以外のディレクトリには書き込まない（dotfiles管理外）
- コードの修正は行わない（ドキュメントとメモリの更新のみ）
- git push は行わない
